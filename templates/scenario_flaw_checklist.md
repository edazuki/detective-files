# 推理シナリオ 欠陥チェックリスト

> シナリオレビュー時に使用する。各項目を確認し、問題があれば備考欄に記録する。

---

## A. 論理的欠陥

### A-1. 犯行の実行可能性

- [ ] 犯人はその時間帯に現場にいられるか（アリバイと矛盾しないか）
- [ ] 犯行に必要な道具・材料を入手できる経路が説明されているか
- [ ] 犯行に必要な体力・技術・知識が犯人の設定と合致するか

### A-2. トリック・カラクリの完全性

- [ ] そのカラクリの目的を説明できるか。カラクリ自体が目的になっていないか。
- [ ] その目的は犯人にとって有益か。
- [ ] その目的を達成する為の、より簡単・安全な方法はないか。それが実行可能な方法であった場合、そうしなかった理由を説明できる証拠はあるか。
- [ ] `truth.md` に犯行手順が**ステップごと**に記述されているか
- [ ] 密室・アリバイ等のカラクリに**複数の要素**がある場合、**それぞれ個別に**実行方法が説明されているか
- [ ] カラクリを実行するための**前提条件**（習慣・構造・道具）が設定内に存在するか
- [ ] カラクリを**解除・逆算したとき**に矛盾が生じないか

### A-3. 第一発見者と密室・現場の整合性 ★

- [ ] 第一発見者は誰か。**犯人と一致する場合**、以下を必ず確認する：
  - [ ] 「現場の状態（施錠・物の位置など）」について、犯人の証言以外の**独立した裏付け**があるか
  - [ ] 犯人が「発見者」として現場に入るためのルートが論理的に説明できるか
  - [ ] 犯人が発見者として現場に入った行為が、自分の仕掛けたカラクリと矛盾しないか
- [ ] 第一発見者が犯人でない場合も、発見者の証言に不自然な点がないか確認する

## B. フェアプレー違反

### B-1. 証拠の到達可能性

- [ ] 解決に必要な全証拠が、プレイヤーが取得可能な形で配置されているか
- [ ] 証拠の取得条件（フラグ・好感度など）が不当に高くないか
- [ ] 証拠の取得条件が循環依存になっていないか（DAGとして到達可能か）
- [ ] 解決の根拠となる情報が、`truth.md` にしか存在していないか確認する（存在するなら証拠化が必要）

### B-2. 物証と証言の区別

- [ ] 重要な事実認定が「証言のみ」に依存していないか
- [ ] 「証言のみ」の場合、その証言者の信頼性は担保されているか
- [ ] **証言者が利害関係者（特に犯人）の場合**、対応する物証が存在するか

### B-3. 特殊知識の要求

- [ ] プレイヤーが一般的な推理で到達できない専門知識が前提になっていないか

## C. 動機・心理の欠陥

### C-1. 動機の妥当性

- [ ] 動機が「その時期・その手段で殺す」ことを説明できるか（タイミングの必然性）
- [ ] より簡単・安全な選択肢がある場合、なぜ複雑な手段を選んだかを説明できる証拠はあるか。
- [ ] 犯行のリスクと動機の利益が釣り合っているか

### C-2. 犯行後の行動

- [ ] 犯行後の犯人の行動（発見者を演じる・捜査への協力など）が心理的に自然か
- [ ] 犯人が「しなければ不自然な行動」をしていない場合、その不作為が説明できるか

---

## D. 時系列・物理的欠陥

### D-1. タイムライン

- [ ] 死亡推定時刻・発見時刻・各証拠の日時が互いに矛盾しないか
- [ ] 全登場人物の行動を時系列で並べたとき、物理的に不可能な移動・行動がないか

### D-2. 物証の存在理由

- [ ] 各物証が「なぜそこにあるか」が説明できるか
- [ ] 犯人以外の人物の持ち物・書類が犯人側から発見される場合、その経緯が説明できるか

### D-3. 表記の一貫性（基本ミス）

- [ ] 登場人物の年齢・名前が全ファイルで一致しているか
- [ ] 日付・曜日・時刻の表記に矛盾がないか

---

## E. 構造的欠陥

### E-1. 犯人の一意性

- [ ] 犯人だけが持つ「他では説明できない証拠との接続」が少なくとも1つあるか

### E-2. 解決の充足性

- [ ] プレイヤーが「犯人・動機・方法」の三要素を全て説明できる証拠が揃っているか
- [ ] 「方法」の説明において、カラクリの各要素（→ A-2 参照）が証拠で裏付けられているか
